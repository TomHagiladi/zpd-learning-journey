# מסע למידה: אזור ההתפתחות המקורב (ZPD)

אפליקציית למידה מונחית + מצגת מלווה למפגש 4 (האחרון) בהכשרת סגל הפקולטה למקצועות הבריאות,
הקריה האקדמית אונו — "הוראה והערכה בעידן הבינה המלאכותית".

## מה יש כאן

| קובץ | תפקיד |
|------|--------|
| `index.html` | אפליקציית מסע הלמידה למשתתפות — 5 תחנות לפי תכנית המפגש |
| `slides.html` | המצגת המלווה של המנחה (כולל QR וטיימרים מובנים) |
| `teacher.html` | לוח מנחה חי — הגשות המשתתפות בזמן אמת (לא מקושר מהאפליקציה) |
| `accessibility.html` | הצהרת נגישות (ת"י 5568 / WCAG 2.0 AA) |

## קישורים חיים

## תשתית

- אתר סטטי על GitHub Pages (ללא build).
- הגשות טקסט נשמרות ב-Firestore — פרויקט `ono-health-gallery` (משותף עם הגלריה של מפגש 2),
  באוספים נפרדים: `zpd_takeaways`, `zpd_products`. חוקים: קריאה והוספה בלבד, עם אימות שדות.
- קבצים (הקלטות, תמונות, שקפים) מוגשים דרך קבוצת הוואטסאפ של ההשתלמות — בחירה מודעת
  שמוסברת למשתתפות בתוך האפליקציה (העלאת קבצים לענן דורשת תוכנית בתשלום).

## התאמה לקהל אחר (תבנית לשימוש חוזר)

הסדנה בנויה כתבנית: מחליפים את **התוכן**, שומרים על **המבנה**.

מה מחליפים בדרך כלל:
1. יחידת הידע (כיום: ZPD) — הכותרת במסך "מפגש עם המושג" וארבעת הפרומפטים.
2. שלושת התרחישים (`scenarios` בראש הסקריפט ב-`index.html`).
3. שאלות הדיון בזוגות (רשימת `q-list`).
4. המחוון (הטבלה בדיאלוג `rubric-dialog` וגם השקף במצגת).
5. שמות האוספים ב-Firestore אם רוצים הפרדה (`zpd_takeaways` / `zpd_products`) + עדכון חוקי האבטחה.
6. יצירת QR חדש אם הכתובת השתנתה.

התיעוד המלא של המפגש נמצא בתיקיית הדרייב של המפגש ובוולט האובסידיאן.
